# Promise

# 基本概念

回调地狱：函数作为参数层层嵌套。

Promise：是异步编程的一种解决方案，主要解决了回调地狱的问题，语法上说，Promise 是一个对象，通过它可以获取异步操作的消息。

then()方法返回的是一个新的Promise实例（不是原来的Promise实例），因此可以采用链式写法，即then方法后面再调用另一个then方法。

catch() 写法可以捕获前面then方法执行中的错误，也更接近同步的写法try/catch。因此建议总是使用catch方法，而不使用 then方法的第二个参数。

Promise 对象的错误具有“冒泡”性质，会一直向后传递，直到被捕获为止，也就是说，错误总是会被下一个catch语句捕获。

特点

1、对象的状态（pending进行中、fulfilled已成功、rejected已失败）不受外界影响； 2、一旦状态改变，就不会再变，任何时候都可以得到这个结果。

缺点

1、一旦新建就会立即执行，无法中途取消； 2、不设置回调函数，Promise内部抛出的错误，不会反应到外部； 3、当处于pending状态时，无法得知目前进展到哪一个阶段。

# 基本使用

const promise = new Promise((resolve, reject) => {
  if (true){
    resolve('success');
  } else {
    reject('error');
  }
});

promise
  .then((data) => {
  	console.log(data);
  }, (error) => {
    console.log(error);
  });

const getJson = function(url) {
  return new Promise(function(resolve, reject){
    const client = new XMLHttpRequest();
    client.open("GET", url);
    client.onreadystatechange = function() {
      if (this.readyState !== 4) {
        return;
      }
      if (this.status === 200) {
        resolve(this.response);
      } else {
        reject(new Error(this.statusText));
      }
    };
    client.responseType = "json";
    client.setRequestHeader("Accept", "application/json");
    client.send();
  });
};

getJson("/posts.json")
  .then((data) => {
    console.log(data);
  })
  .catch((error) => {
    console.log(error);
  })
  .finally(() => {
    // 不接受参数，不知道前面 Promise 的状态；
    // 不管状态，执行完 then 和 catch 后都会执行 finally 的回调函数；
  });

# 方法

.then()：指定resolved和rejected状态的回调函数； .catch()：指定发生错误时的回调函数； .finally()：指定不管最后状态如何都会执行的回调函数； Promise.race()：将多个实例包装成一个新实例，返回全部实例状态优先变更后的结果（先变更先返回）； Promise.any()：将多个实例包装成一个新实例，返回全部实例状态变更后的结果数组(齐变更再返回) Promise.all()：将多个实例包装成一个新实例，返回全部实例状态变更后的结果数组（齐变更再返回）； Promise.allSettled()：将多个实例包装成一个新实例，返回全部实例状态变更后的状态和结果数组(齐变更再返回)； Promise.try()：不想区分是否同步异步函数，包装函数为实例，使用then()指定下一步流程，使用catch()捕获错误； Promise.resolve()：将对象转为Promise对象，new Promise(resolve => resolve())； Promise.reject()：将对象转为状态为rejected的Promise对象，new Promise((resolve, reject) => reject())；

# Class

# 基本概念

ES6 的class可以看作只是一个语法糖，它的绝大部分功能，ES5 都可以做到，新的class写法只是让对象原型的写法更加清晰、更像面向对象编程的语法而已。

类：对一类具有共同特征的事物的抽象（构造函数语法糖）。

类的数据类型就是函数，类本身就指向构造函数（Class === Class.prototype.constructor）；

类的所有实例共享一个原型对象（c1.__proto__ === c2.__proto__）；

类的实例调用方法，就是调用原型上的方法；

类的内部所有定义的方法，都是不可枚举的；

类的实例：

显式指定属性方法：使用this指定到自身上，Class.hasOwnProperty() 可检测到；

隐式指定属性方法：直接声明定义在对象原型上，Class.__proto__.hasOwnProperty() 可检测到；

类的属性：

Class.__proto__：指向父类；

Class.__proto__.__proto__：指向父类的__proto__；

Class.prototype.__proto__：指向父类的prototype；

# 基本使用

class Point {
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }

  toString() {
    return `${this.x}, ${this.y}`;
  }
}

const p = new Point();
p.toString(10, 11);

typeof Point // "function"
Point === Point.prototype.constructor // true

class Point {
  constructor() {}
  toString() {}
}

// 等同于

Point.prototype = {
  constructor() {},
  toString() {},
};

// 继承 extends
class Point extends Geometry {
  constructor(x, y) {
    super(); // 父类静态属性方法可被子类继承，先将父类实例的属性方法加到 this 上，再用子类构造函数修改 this。
  }
}
